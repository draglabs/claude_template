# Execution incidents

Problems with the SOP — policy violations, briefing errors, process gaps encountered during execution. Each incident should result in a policy fix so it doesn't repeat.

Not for code bugs — those go in issue tracking or a separate bugs doc.

## Template

```markdown
### EI-NNN: Short title
- **When:** date/context
- **What:** what went wrong
- **Root cause:** why it happened
- **Impact:** Low / Medium / High — what was lost (time, work, correctness)
- **Fix:** what policy/process change prevents recurrence
- **Status:** Active (monitoring) / Resolved
```
